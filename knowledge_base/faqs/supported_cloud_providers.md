# Supported Cloud Providers

## Overview
Clouddash provides native integration and support for the following cloud platforms and services. We continuously expand our provider coverage based on customer demand.

## Fully Supported Cloud Providers

### Amazon Web Services (AWS)
**Coverage**: Comprehensive support for AWS services
- EC2, ECS, Lambda monitoring
- CloudWatch metrics and logs
- IAM and credential management
- VPC and networking resources
- RDS and database services
- S3 and storage services

**Features**:
- Real-time metric ingestion
- Log aggregation
- Alert routing
- Custom dashboard creation

### Google Cloud Platform (GCP)
**Coverage**: Full GCP integration
- Compute Engine instances
- Cloud Run services
- Stackdriver monitoring
- Cloud Storage metrics
- BigQuery monitoring

**Features**:
- Native API authentication
- Automatic resource discovery
- Custom metric visualization

### Microsoft Azure
**Coverage**: Comprehensive Azure support
- Virtual Machines
- App Services
- Azure SQL Database
- Azure Storage
- Application Insights integration

**Features**:
- Service Principal authentication
- Resource group organization
- Cost management metrics

## Limited Support
- **Kubernetes (EKS, GKE, AKS)**: Metrics and log collection available through cloud provider integrations
- **On-Premises**: Limited support through API endpoints and custom integrations

## Not Currently Supported
- Private cloud platforms (OpenStack, CloudStack)
- Legacy hosting providers
- Proprietary monitoring tools (we do not provide Datadog integration)

## Requesting New Provider Support
If you need integration with a provider not listed above:
1. Contact your account manager
2. Submit a feature request through the support portal
3. Include your use case and timeline requirements

## Migration from Other Platforms
If migrating from another monitoring platform:
- We provide data import utilities for supported formats
- Reach out to our onboarding team for assistance
- Typical migration takes 1-2 weeks
